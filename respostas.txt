1- Integração com a API do ViaCEP:
Ao preencher o campo CEP e sair dele, é feita uma requisição fetch para a URL https://viacep.com.br/ws/{CEP}/json/, 
que retorna o endereço completo para preencher automaticamente o formulário.

2-Geração da imagem aleatória:
Usamos a API do Unsplash para buscar uma foto aleatória com o tema "voluntário" usando a chave de acesso (access key) e exibimos essa imagem em cada card.

3-Uso do LocalStorage:
Os dados dos voluntários e administradores são salvos e recuperados do LocalStorage, garantindo que os dados persistam mesmo após a atualização da página.

4-Organização dos cards:
Os voluntários são exibidos em cards dentro de um container, que inclui foto, nome, email, endereço e um botão de exclusão.

5-Sistema de login e controle de acesso:
O login valida o email e senha contra os dados armazenados no LocalStorage. Após login válido, o usuário é redirecionado ao menu.

6-Controle de e-mail duplicado:
No cadastro do administrador, antes de salvar o sistema verifica se o email já existe no LocalStorage e impede que o cadastro seja duplicado.

7-Redirecionamento após 5 minutos de inatividade:
Um timer é resetado a cada ação do usuário movimento do mouse, tecla, clique, scroll, e após 5 minutos sem atividade, o sistema alerta e redireciona para a página de login.

8-O que faria diferente em um projeto real?
Implementaria backend com banco de dados para armazenar dados com segurança.
